# 00 - LED Blink Baseline

Simple LED blink test for the STM32F407 Discovery board.

The current code uses the onboard green LED `LD4` on port `GPIOD`, pin `12`.

## Goal

Make sure the board, flash/debug path, and basic GPIO output are all working before moving on to larger examples.

## What the added loop does

Inside `while (1)`, the code reads the current system tick with `HAL_GetTick()` and compares it with the last time the LED changed state.

```c
uint32_t now = HAL_GetTick();

if (now - last_toggle_time_ms >= led_toggle_interval_ms) {
  HAL_GPIO_TogglePin(LD4_GPIO_Port, LD4_Pin);
  last_toggle_time_ms = now;
}
```

This is a non-blocking blink pattern. The loop keeps running, and the LED only toggles when 500 ms have passed. There is no `HAL_Delay()` in the loop, so the CPU is free to keep servicing other work between toggles.

## Variables

- `led_toggle_interval_ms`: blink period in milliseconds
- `last_toggle_time_ms`: tick value captured at the last LED toggle
- `now`: current tick value returned by `HAL_GetTick()`

## Why `LD4_GPIO_Port` and `LD4_Pin`

Those names come from the CubeMX-generated board configuration in `main.h`. Using them is better than hardcoding the port and pin in the loop because the intent stays clear and the code matches the board setup.

## Notes

- Most of `main.c` was generated by CubeMX.
- For this baseline, the blink logic is the part to focus on.
- The project still contains extra peripheral initialization from the generated template, but the LED timing code itself is separate from that.
